# APP NOTES

Файл фиксирует текущее состояние проекта по коду, который находится в рабочей папке. Проект использует `.git` и синхронизирован с репозиторием на GitHub.

## Зафиксированные изменения

1. Основной интерфейс приложения сейчас реализован в `app.py`, а не в Qt-версии.
2. Текущий GUI построен на `tkinter` и рисуется через `Canvas`, карточки и собственную прокрутку.
3. В заголовок окна добавлен build-маркер `CANVAS BUILD 2026-04-28-01`.
4. Приложение поддерживает три типа источника: один PDF, папка с PDF, ZIP-архив с PDF.
5. После выбора источника анализ запускается автоматически без отдельного подтверждения.
6. Анализ вынесен в фоновый поток, чтобы интерфейс не зависал во время обработки PDF.
7. Результаты анализа передаются обратно в UI через очередь `analysis_queue`.
8. В интерфейсе показываются:
   - выбранный источник
   - счетчики по инвойсам, общему весу и сумме USD
   - текстовая сводка результата
   - таблица по инвойсам
   - лог и строка статуса
9. Для ZIP-архивов добавлена распаковка во временную директорию с последующей очисткой.
10. В Excel-экспорт добавлены `Origin Countries`, итоговый список стран происхождения, общий вес и общая сумма.
11. Парсер умеет работать через `PyPDF2` и fallback через системный `PDFKit`.
12. Если два движка извлечения текста дают разные результаты, приложение выбирает более сильный вариант и фиксирует предупреждение.
13. Для дублей используется правило "первый найденный `Invoice No` остается, следующий исключается".
14. В `Notes` Excel-отчета сохраняются дубликаты, ошибки парсинга и предупреждения.
15. `README.md` обновлен под текущее приложение, чтобы документация соответствовала реальному коду.
16. Проект переведен на кроссплатформенные диалоговые окна выбора файлов (`tkinter.filedialog`) вместо macOS-специфичного `osascript`.
17. Файл `parser.py` переименован в `invoice_parser.py` для исключения конфликта имен со встроенным модулем Python на Windows.
18. Настроена автоматическая сборка Windows-версии через GitHub Actions (`.github/workflows/build-windows.yml`).
19. Добавлен `.gitignore` для исключения виртуального окружения, временных файлов сборки и логов из репозитория.

## Основные правила приложения

1. Перед анализом обязательно должен быть выбран источник данных.
2. Источник может быть только одного из трех типов: PDF, папка, ZIP.
3. Из папки и ZIP обрабатываются только файлы с расширением `.pdf`.
4. Новый анализ нельзя запускать параллельно с уже идущим анализом.
5. Успешным считается только тот инвойс, у которого найдены:
   - `Invoice No`
   - `Total USD`
   - хотя бы один вес позиции
6. Дубликаты определяются только по `Invoice No`.
7. При дубле в итоговом списке остается первый найденный файл, остальные помечаются как исключенные.
8. Если PDF не распознан, файл попадает в замечания и не включается в итоговые данные.
9. Если в выбранном источнике нет PDF, приложение показывает отдельное сообщение об этом.
10. Если PDF обработаны, но инвойсы Cummins не найдены, приложение сообщает об этом в результате и логе.
11. Таблица в окне приложения может показывать только часть строк, но экспорт в Excel должен содержать все найденные инвойсы.
12. При очистке состояния приложение сбрасывает результаты, лог, источник и временные файлы.
13. При закрытии окна временные файлы также должны быть удалены.

## Практические правила для дальнейших изменений

1. Любое изменение логики анализа должно сохранять совместимость с выбором PDF, папки и ZIP.
2. Любое изменение парсинга должно учитывать оба режима извлечения текста: `PyPDF2` и `PDFKit`.
3. Любое изменение UI должно сохранять видимость статуса, результатов, таблицы и лога.
4. Любое изменение Excel-экспорта должно не ломать листы `Summary` и `Notes`.
5. Если меняется реальная точка входа или GUI-технология, это нужно сразу отражать в `README.md`.
