# Cummins Invoice Studio

Desktop-приложение на Python для анализа PDF-инвойсов Cummins, проверки дублей и экспорта итогового отчета в Excel.

## Актуальное состояние

- текущая рабочая версия интерфейса находится в `app.py`
- интерфейс реализован на `tkinter` с canvas-layout и прокруткой
- приложение работает с одним PDF, папкой PDF или ZIP-архивом с PDF
- после выбора источника анализ запускается автоматически
- результаты выводятся на экране и могут быть сохранены в `.xlsx`

Сборочные и вспомогательные файлы для прежних вариантов интерфейса в проекте сохранены, но актуальная логика запуска сейчас описана именно для `app.py`.

## Структура проекта

- `app.py` — актуальное GUI-приложение и точка входа
- `parser.py` — парсинг PDF-инвойсов Cummins
- `exporter.py` — экспорт `Summary` и `Notes` в Excel
- `utils.py` — выбор файлов, работа с ZIP и форматирование
- `analyze_invoices.py` — запуск анализа без GUI
- `verify_release.py` — проверка на тестовых наборах
- `qt_app.py` — прежняя Qt-версия, оставлена в проекте
- `sample_data/` — тестовые примеры
- `assets/` — иконки

## Что умеет приложение

- выбрать один PDF-файл
- выбрать папку с PDF-файлами
- выбрать ZIP-архив с PDF-файлами
- извлечь `Invoice No`, веса по позициям, общий `Net Weight` и `Total USD`
- определить страны происхождения из PDF-текста
- исключить дубликаты по `Invoice No`
- показать итоги, таблицу и лог прямо в окне приложения
- сохранить Excel-отчет

## Основные правила работы приложения

1. Источник данных должен быть выбран до запуска анализа.
2. Поддерживаются только PDF-файлы, даже если источник выбран как папка или ZIP.
3. После выбора PDF, папки или ZIP анализ стартует сразу.
4. Если анализ уже идет, повторный запуск не начинается.
5. Дубликатом считается повторный `Invoice No`; сохраняется первый найденный инвойс, следующий исключается.
6. Инвойс считается успешно распознанным только если найдены `Invoice No`, `Total USD` и хотя бы один вес позиции.
7. Если `PyPDF2` не дал хороший текст, приложение использует fallback через `PDFKit` на macOS.
8. Если оба движка дали результат, приложение выбирает более полный и пишет предупреждение при расхождении.
9. В таблице интерфейса показывается только часть строк, но в Excel попадают все распознанные инвойсы.
10. При очистке состояния временные файлы от распакованного ZIP удаляются.

## Экспорт в Excel

Формируется файл `.xlsx` с двумя листами:

- `Summary` — инвойсы, позиции, страны происхождения, веса и суммы
- `Notes` — дубликаты, ошибки и предупреждения

В `Summary` дополнительно считаются:

- `ALL Origin Countries`
- `TOTAL Net Weight`
- `TOTAL USD`

## Логика парсинга

- `Invoice No` ищется по блоку `Invoice No:`
- `Total USD` ищется по `Order Total ... USD` или `USD TO PAY`
- веса берутся из товарных строк рядом с секцией `Custom Stat No.`
- если стандартная структура не сработала, используется compact fallback для извлечения весов

Инвойс не попадает в итоговый список, если:

- не найден номер инвойса
- не найдена сумма `Total USD`
- не найден ни один вес позиции
